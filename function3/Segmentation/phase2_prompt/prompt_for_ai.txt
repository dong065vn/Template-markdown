# IV. Nội dung chi tiết từng giai đoạn



## Tuần 1–2: Lý thuyết cơ bản


Mục tiêu: Hiểu hệ thống tổng thể trước khi động vào thực hành.

Nội dung:

Khái niệm Cloud Computing, Edge Computing, Fog Computing.

Mô hình dịch vụ: IaaS – PaaS – SaaS.

Sự khác biệt giữa Cloud và Edge.

Các ví dụ thực tế: Digital signage, order app, smart campus.

Đầu ra: Slide "Tổng quan Edge & Cloud".


## Tuần 3: Làm quen thiết bị Edge


Thiết bị có thể dùng:

WENet Services Point

Kết nối mạng LAN/Wi-Fi, SSH remote.

Cấu hình thời gian, IP tĩnh, cập nhật hệ thống.

Cài thử Docker.

Đầu ra: Logbook cấu hình + video ngắn minh họa SSH/Edge hoạt động.


## Tuần 4–5: Thực hành theo mô hình WENet Cloud–Edge


Mục tiêu:

Giúp sinh viên hiểu luồng hoạt động thực tế của hệ thống Edge–Cloud trong môi trường: Smart Campus, Digital Signage, Order App, Event App, Tour App.

Biết cấu hình thiết bị Edge, truyền dữ liệu lên Cloud Edge, quan sát tương tác người dùng (qua smartphone hoặc màn hình signage).

Làm quen với cách ứng dụng edge node trong hệ thống phân tán.

Bài thực hành mẫu:


### Mẫu bài thực hành 1: Digital Signage thông minh


Mục tiêu: Hiểu cách edge xử lý và phân phối nội dung hiển thị mà không phụ thuộc hoàn toàn vào cloud.

Công việc chi tiết:

Cấu hình một Edge node (WENet Service Point).

Tạo playlist video / hình ảnh (HTML).

Cloud chỉ gửi lịch hiển thị hoặc nội dung cập nhật định kỳ.

Khi mất kết nối cloud, Edge vẫn hiển thị nội dung.


### Mẫu bài thực hành 2: Smartphone Interactive Display


Mục tiêu: Trải nghiệm cách người dùng smartphone tương tác với thiết bị edge.


### Mẫu bài thực hành 3: Smart Campus Dashboard


Mục tiêu: Xây dựng bảng điều khiển thông minh (dashboard) hiển thị dữ liệu từ các Edge node trong khuôn viên (phòng học, cổng ra vào).

Các bài thực hành tương tự:

Event App

Tour App

Order App

Đầu ra: Báo cáo kiến trúc hệ thống + Biểu đồ kết nối Edge-Cloud + Nguyên lý vận hành.
